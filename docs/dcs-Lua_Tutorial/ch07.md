## How it is used in DCS

In DCS mission scripting, a `CFG` table is perfect for:

- Group names (`"TGT_01"`, `"TGT_02"`)
- Message durations
- F10 menu texts
- Timer intervals

Simple idea:

- **CFG**: things you change (configuration)
- **Logic**: what the mission does

## Common errors and fixes

- **`CFG.welcome` doesn‚Äôt work**:
  - Confirm you created `local CFG = { ... }` above.
  - Confirm commas inside the table.
- **Braces/commas errors**:
  - The most common mistake is a missing comma at the end of a line.

> **If you get an error, do this:**
> - Keep the table with one field only:
>   - `local CFG = { welcome = "Hello" }`
> - When it works, add more fields.

## Mini‚Äëlab (15‚Äì30 min): a target list (config only)

Goal: create a list of target group names in `CFG` (we will activate them later).

1) Create `my_ex07.lua`.
2) Paste:

```lua
local CFG = {
  targets = { "TGT_01", "TGT_02", "TGT_03" }
}

trigger.action.outText("Targets: " .. table.concat(CFG.targets, ", "), 10)
```

3) Run and confirm you see the names.

## Checklist (what you must see in-game)

- Message shows ‚ÄúTargets: TGT_01, TGT_02, TGT_03‚Äù.
- Changing the list changes the message without touching logic.
