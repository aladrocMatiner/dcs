## How it is used in DCS

In a mission, variables let you:

- Store ‚Äústate‚Äù (attempts, points, phases).
- Store small configuration (message durations).
- Avoid repeating the same numbers everywhere.

In Lua you‚Äôll commonly write:

- `local name = "Pilot"` (text)
- `local attempts = 1` (number)
- `local training = true` (boolean)

`local` means: ‚Äúkeep this variable here, don‚Äôt pollute the whole mission‚Äù.

## Common errors and fixes

- **I see `nil` or weird output**:
  - Text needs quotes: `"Pilot"`.
  - Lua concatenates with `..` (not `+`).
- **No log line**:
  - Find `dcs.log` and search for `DCS-Lua-Tutorial`.

> **If you get an error, do this:**
> - Reduce to:
>   - `local x = 1`
>   - `trigger.action.outText("x=" .. x, 10)`
> - When that works, add more lines.
